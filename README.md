# Provenance Guard

Provenance Guard is a Flask backend API that analyzes submitted text to
determine whether it is more likely **written by a human** or **generated by
AI**.

This repository currently implements **Milestone 3**:

- Flask backend with `GET /`, `POST /submit`, and `GET /log`
- Groq large language model as the first detection signal
- Structured audit logging to a JSON file
- UUID-based content IDs

Stylometric heuristics, combined confidence scoring, appeals, transparency
labels, and rate limiting are planned for later milestones.

## Installation

### 1. Create and activate a virtual environment

```bash
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

### 2. Install requirements

```bash
pip install -r requirements.txt
```

### 3. Create your `.env`

Copy the example file and add your Groq API key:

```bash
cp .env.example .env
```

```
GROQ_API_KEY=your_key_here
```

## Run

```bash
python app.py
```

The API starts on `http://localhost:5000`.

## API

### `GET /`

Health check.

**Response**

```json
{
  "status": "running"
}
```

### `POST /submit`

Classify a block of text.

**Request body**

```json
{
  "text": "Some text...",
  "creator_id": "user123"
}
```

**Response**

```json
{
  "content_id": "6f1c2e5a-8d3b-4a2f-9c11-2b7d0e4a1f33",
  "attribution": "Likely Human",
  "confidence": 0.82,
  "label": "Placeholder label (Milestone 4)",
  "status": "classified"
}
```

### `GET /log`

Return every recorded audit entry.

**Response**

```json
{
  "entries": [
    {
      "content_id": "6f1c2e5a-8d3b-4a2f-9c11-2b7d0e4a1f33",
      "creator_id": "user123",
      "timestamp": "2026-07-01T22:10:00Z",
      "attribution": "Likely Human",
      "confidence": 0.91,
      "llm_score": 0.91,
      "status": "classified"
    }
  ]
}
```

## Testing with curl

### Health check

```bash
curl http://localhost:5000/
```

### Submit text

```bash
curl -X POST http://localhost:5000/submit \
  -H "Content-Type: application/json" \
  -d '{"text": "The quick brown fox jumps over the lazy dog.", "creator_id": "user123"}'
```

### View the audit log

```bash
curl http://localhost:5000/log
```

## Error handling

| Situation                | HTTP status | Body                                     |
| ------------------------ | ----------- | ---------------------------------------- |
| Invalid JSON body        | `400`       | `{"error": "Invalid JSON body"}`         |
| Missing `text`           | `400`       | `{"error": "Field 'text' is required"}`  |
| Missing `creator_id`     | `400`       | `{"error": "Field 'creator_id' is required"}` |
| Groq API failure         | `500`       | `{"error": "Groq service unavailable"}`  |
