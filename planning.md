# Provenance Guard

## Project Overview

Provenance Guard is a backend API that helps determine whether submitted text is more likely written by a human or generated by AI. The system combines multiple independent detection signals, calculates a confidence score, generates a transparency label for users, stores every attribution decision in an audit log, and provides an appeals workflow for creators who believe their work was incorrectly classified.

The goal is not to provide perfect AI detection—an unsolved research problem—but to build a trustworthy and transparent attribution system that communicates uncertainty honestly while giving creators a mechanism to challenge incorrect decisions.

---

# Architecture Narrative

## Submission Flow

When a user submits a piece of text, the request is sent to the `POST /submit` endpoint.

The system first validates the request to ensure the submitted text is not empty and meets basic formatting requirements.

The validated text is then passed to two independent detection systems:

1. **Groq LLM Classification**
   - Uses the Groq API with the `llama-3.3-70b-versatile` model.
   - Determines whether the writing appears more likely human-written or AI-generated.
   - Returns a prediction and confidence estimate.

2. **Stylometric Heuristic Analysis**
   - Computes measurable writing statistics including:
     - Sentence length variance
     - Average sentence length
     - Type-token ratio (vocabulary diversity)
     - Punctuation density
   - Produces a heuristic score representing how closely the writing resembles typical human writing patterns.

Both outputs are sent to the Confidence Calculator.

The Confidence Calculator combines the two signals into a single confidence score and final attribution decision.

The Transparency Label Generator converts the prediction into a human-readable message suitable for display on a content platform.

Finally, the complete decision—including both detection signals, confidence score, transparency label, and timestamp—is stored in the audit log before the JSON response is returned to the user.

---

## Appeal Flow

If a creator believes the system incorrectly classified their work, they may submit an appeal through the `POST /appeal` endpoint.

The appeal includes:

- Content ID
- Creator explanation

The system records the appeal, updates the content status to **Under Review**, logs the appeal inside the audit log, and returns confirmation to the creator.

No automatic reclassification occurs during the appeal process.

# Detection Signals

## Signal 1 — Groq LLM Classification

### Purpose

Uses the Groq `llama-3.3-70b-versatile` model to determine whether submitted text appears more likely to be human-written or AI-generated.

### Measures

- Semantic coherence
- Logical flow
- Writing style
- Tone
- Language consistency

### Output

```text
Prediction:
Human
or
AI

Confidence:
0.00–1.00
```

### Weight

60%

### Blind Spot

The LLM may incorrectly classify polished human writing as AI-generated or sophisticated AI writing as human-written.

---

## Signal 2 — Stylometric Heuristics

### Purpose

Measures statistical properties of the writing.

### Metrics

- Sentence length variance
- Average sentence length
- Type-token ratio
- Punctuation density

### Output

```text
Stylometric Score

0.00–1.00
```

### Weight

40%

### Blind Spot

Professional human writers and advanced AI systems can produce similar statistical writing patterns.

---

## Combining Signals

The final confidence score is calculated using weighted averaging.

```
Final Score

=
(Groq Confidence × 0.60)

+

(Stylometric Score × 0.40)
```

The final prediction is based on the combined score rather than either signal individually.

## Signal 1 — Groq LLM Classification

### Description

The Groq Large Language Model analyzes the submitted text using semantic reasoning.

### What It Measures

- Logical coherence
- Writing style
- Semantic consistency
- Organization
- Natural language fluency

### Why It Was Chosen

Large Language Models capture high-level language characteristics that are difficult to measure using simple statistics.

### Blind Spot

The model may incorrectly classify exceptionally polished human writing as AI-generated or sophisticated AI-generated writing as human-written.

Therefore, this signal alone should never determine the final decision.

---

## Signal 2 — Stylometric Heuristics

### Description

The stylometric analyzer measures structural writing characteristics using deterministic Python code.

### Metrics

- Sentence length variance
- Average sentence length
- Type-token ratio
- Punctuation density

### Why It Was Chosen

Human writing generally exhibits greater variation than AI-generated writing. Statistical analysis provides an independent signal that complements the semantic reasoning of the LLM.

### Blind Spot

Professional authors often write consistently, while modern AI systems can intentionally introduce variability.

Stylometric analysis alone cannot reliably identify AI-generated writing.

---

# Multi-Signal Decision Strategy

The final classification is based on both detection signals.

The Groq classifier provides semantic understanding while the stylometric analyzer provides structural evidence.

Combining independent signals reduces reliance on any single detection method and produces a more reliable confidence score.

---

# Confidence Scoring Strategy

Confidence scores range from **0.00 to 1.00**.

| Confidence | Interpretation      |
| ---------- | ------------------- |
| 0.90–1.00  | High Confidence     |
| 0.70–0.89  | Moderate Confidence |
| Below 0.70 | Uncertain           |

The confidence score communicates how certain the system is rather than pretending every prediction is definitive.

This design reduces false positives and encourages cautious interpretation.

# Uncertainty Representation

The system communicates uncertainty instead of forcing every submission into a binary AI or Human decision.

## Confidence Meaning

| Score       | Interpretation       |
| ----------- | -------------------- |
| 0.90 – 1.00 | Very High Confidence |
| 0.75 – 0.89 | High Confidence      |
| 0.60 – 0.74 | Moderate Confidence  |
| Below 0.60  | Uncertain            |

A confidence score of **0.60** means the two detection signals disagree or neither signal provides strong evidence.

Rather than claiming the content is AI-generated, the system displays an Uncertain transparency label.

## This reduces the likelihood of false positives.

# False Positive Analysis

False positives represent the most harmful failure mode because they incorrectly label human-created work as AI-generated.

Example:

A human author submits an original short story.

The Groq classifier predicts AI-generated with moderate confidence while the stylometric analysis produces mixed evidence.

Instead of displaying a definitive AI label, the system returns an Uncertain transparency label with a confidence score around 0.60.

The creator may then submit an appeal explaining why the work is original.

The appeal is stored in the audit log and the content status changes to **Under Review**.

This workflow protects creators from unfair automated decisions.

# Appeals Workflow

Any creator whose content has been analyzed may submit an appeal.

The appeal must contain:

- Content ID
- Creator explanation

When an appeal is submitted, the system will:

1. Store the appeal.
2. Update the content status to **Under Review**.
3. Record the appeal inside the audit log.
4. Return confirmation to the creator.

Human reviewers will see:

- Original text
- Groq prediction
- Stylometric score
- Final confidence
- Transparency label
- Appeal reason
- Current review status

```
              Creator
                  │
                  ▼
            POST /appeal
                  │
                  ▼
         Validate Appeal Request
                  │
                  ▼
      Update Status → Under Review
                  │
                  ▼
            Update Audit Log
                  │
                  ▼
            JSON API Response
```

# Rate Limiting Strategy

The submission endpoint will be limited to:

**10 requests per minute per IP address**

This limit supports normal platform usage while reducing the risk of automated abuse or denial-of-service attacks.

---

# Audit Log Design

Each audit entry stores:

- Timestamp
- Submitted text
- Groq prediction
- Stylometric score
- Combined confidence score
- Final prediction
- Transparency label
- Appeal status
- Appeal reason (if submitted)

The audit log provides traceability for every attribution decision.

# Anticipated Edge Cases

## Edge Case 1

A short poem that intentionally repeats words and uses simple vocabulary.

The stylometric analysis may incorrectly classify it as AI-generated because repetitive writing often resembles AI output.

---

## Edge Case 2

A professional technical article written by an experienced human author.

The writing may be highly structured and consistent, causing the Groq classifier to assign a higher probability of AI generation.

---

## Edge Case 3

AI-generated writing that has been heavily edited by a human.

The semantic signal may detect AI patterns while the stylometric analysis resembles human writing.

The combined score may therefore produce an Uncertain result.
